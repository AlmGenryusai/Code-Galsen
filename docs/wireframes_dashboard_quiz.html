Specs d'implémentation.

Tous les composants utilisent les CSS variables du système. Pas de framework UI lourd au-delà de shadcn/ui pour les primitives (Button, Sheet, Dialog).

Dashboard — comportement

  • État A si user_progress vide → CTA "Choisir un thème" mène à /themes.
  • État B si pass actif + au moins 1 thème commencé.
  • État C si streak_days >= 3. Streak calculé par requête locale Dexie.
  • Pass badge en haut à droite : décompte jours via pass_jwt.exp - now. Couleur passe à --terra si < 7 jours.
  • Cartes "À réviser" = countDue(srs_queue). "ExamScore" = moyenne 5 derniers quiz_attempts.

Quiz — comportement

  • Timer 20s/question : barre de progression linéaire + texte. Passe en rouge à 5s restants.
  • Tap réponse → désactive autres options (état dim), affiche feedback strip 1,5s, auto-next.
  • Si faute : enregistre dans srs_queue via initSrs() (cf 14_srs.ts).
  • Examen blanc : compteur fautes visible. À 5 fautes → écran "Échec" (variant état D rouge).
  • Audio Wolof standby V1 : badge "FR" uniquement. Réintroduction V2 avec badge "WO".
  • Fin examen : push quiz_attempts + quiz_answers avec synced=0. Background Sync API tente l'upload.

Composants partagés

  • <PhoneFrame> wrapper conteneur 100vh mobile.
  • <ScoreCard> variants default | warning | success.
  • <ThemeRow> + <ThemeRowSkeleton>.
  • <QuizOption> états idle | dim | correct | wrong.
  • <FeedbackStrip> variants success | error avec auto-dismiss.
